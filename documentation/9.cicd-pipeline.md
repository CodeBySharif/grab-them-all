# 9. CI/CD Pipeline

Continuous Integration / Continuous Deployment is automated in **GitHub Actions**.

**File:** `.github/workflows/deploy.yml`

---

## Trigger

```yaml
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
```

| Event | Build job | Deploy job |
|---|---|---|
| Pull request to `main` | ✅ Runs | ❌ Skipped |
| Push to `main` | ✅ Runs | ✅ Runs |

> **Best practice:** Run build on PRs to catch errors before merge. Deploy only on push to `main`.

---

## Pipeline overview

```
┌─────────────┐     ┌─────────────┐
│  build job  │────►│ deploy job  │   (main push only)
└─────────────┘     └─────────────┘
       │                    │
       ├─ dotnet build      ├─ auth GCP
       ├─ npm ci + build    ├─ docker build API → push
       │                    ├─ deploy ferry-flight-api
       │                    ├─ docker build web → push
       │                    ├─ deploy ferry-flight-web
       │                    ├─ update API CORS
       │                    └─ print URLs
```

---

## Workflow environment variables

Defined at top of `deploy.yml`:

```yaml
env:
  GCP_REGION: asia-southeast1
  AR_REPOSITORY: ferry-flight
  API_SERVICE: ferry-flight-api
  WEB_SERVICE: ferry-flight-web
  PUBLIC_WEB_URL: https://grab-them-all.shariflab.my
  PUBLIC_API_URL: https://grab-them-all-api.shariflab.my
```

Change `PUBLIC_WEB_URL` and `PUBLIC_API_URL` when your domains change.

---

## Job 1: `build`

Validates both apps compile before deploy.

### Backend step

```yaml
- uses: actions/setup-dotnet@v4
  with:
    dotnet-version: '10.0.x'
- run: dotnet build backend/FerryFlight.Api/FerryFlight.Api.csproj -c Release
```

### Frontend step

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: '22'
    cache: npm
- working-directory: frontend
  run: |
    npm ci
    npm run build
```

`npm run build` also runs icon generation and TypeScript check.

---

## Job 2: `deploy`

Runs only when: `github.event_name == 'push' && github.ref == 'refs/heads/main'`

### Step-by-step

| Step | What happens | Files involved |
|---|---|---|
| 1. Checkout | Clone repo | — |
| 2. Authenticate GCP | Uses `secrets.GCP_SA_KEY` | Service account JSON |
| 3. Setup gcloud | Install Cloud SDK | — |
| 4. Configure Docker | Auth to Artifact Registry | — |
| 5. Ensure registry | Create `ferry-flight` repo if missing | — |
| 6. Build & push API | `docker build` + `docker push` | `backend/FerryFlight.Api/Dockerfile` |
| 7. Deploy API | `gcloud run deploy ferry-flight-api` | Sets env vars, memory, port |
| 8. Build & push Web | `docker build` + `docker push` | `frontend/Dockerfile` |
| 9. Deploy Web | `gcloud run deploy ferry-flight-web` | Sets `BACKEND_URL=${PUBLIC_API_URL}` |
| 10. Update CORS | `gcloud run services update` | `Cors__Origins__0=${PUBLIC_WEB_URL}` |
| 11. Print URLs | Echo custom + Cloud Run URLs | — |

## Docker image tags

Each deploy tags images with the Git commit SHA:

```
asia-southeast1-docker.pkg.dev/PROJECT/ferry-flight/api:abc123def
asia-southeast1-docker.pkg.dev/PROJECT/ferry-flight/web:abc123def
```

Immutable tags make rollbacks traceable.

---

## API deploy details

```bash
gcloud run deploy ferry-flight-api \
  --image "$IMAGE" \
  --region asia-southeast1 \
  --allow-unauthenticated \
  --port 8080 \
  --memory 512Mi \
  --set-env-vars "ASPNETCORE_ENVIRONMENT=Production[,OpenRouteService__ApiKey=...]"
```

OpenRouteService key is included only if `OPENROUTESERVICE_API_KEY` GitHub secret exists.

---

## Web deploy details

```bash
gcloud run deploy ferry-flight-web \
  --image "$IMAGE" \
  --set-env-vars "BACKEND_URL=${PUBLIC_API_URL}"
```

At container start, `docker-entrypoint.sh` reads `BACKEND_URL` and generates nginx config.

---

## GitHub secrets required

| Secret | Required | Purpose |
|---|---|---|
| `GCP_PROJECT_ID` | Yes | Target GCP project |
| `GCP_SA_KEY` | Yes | Service account JSON for deploy |
| `OPENROUTESERVICE_API_KEY` | No | Road travel estimates in API |

## What happens inside the web Docker build

```
frontend/Dockerfile
  Stage 1 (node:22-alpine):
    npm ci
    npm run build          ← Vite + PWA + tsc
  Stage 2 (nginx:1.27-alpine):
    COPY dist/ → /usr/share/nginx/html
    COPY nginx.conf.template, docker-entrypoint.sh
    CMD docker-entrypoint.sh → nginx
```

---

## What happens inside the API Docker build

```
backend/FerryFlight.Api/Dockerfile
  Stage 1 (dotnet/sdk:10.0):
    dotnet restore + publish
  Stage 2 (dotnet/aspnet:10.0):
    COPY publish output
    ENTRYPOINT dotnet FerryFlight.Api.dll
    Listens on PORT env (8080)
```

## Common CI/CD failures

| Error | Cause | Fix |
|---|---|---|
| Docker push denied | SA lacks `artifactregistry.writer` | Re-run IAM binding from doc 8 |
| Cloud Run deploy denied | SA lacks `run.admin` | Re-run IAM binding |
| `npm run build` fails | TypeScript error | Fix locally, push again |
| `dotnet build` fails | C# compile error | Fix locally, push again |
| Web `/api` 404 after deploy | Wrong `BACKEND_URL` or nginx Host header | Check workflow env + `docker-entrypoint.sh` |

## Best practices used

- **Build before deploy** — broken code never reaches Cloud Run
- **Immutable image tags** — one image per commit
- **Secrets in GitHub** — never in repo or Dockerfile
- **Env vars in workflow** — custom domains survive redeploys
- **PR builds** — validate without deploying
