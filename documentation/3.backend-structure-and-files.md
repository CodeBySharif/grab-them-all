# 3. Backend Structure and Files

Backend path: `backend/FerryFlight.Api/`

The backend is a single ASP.NET Core **Web API** project. It has no database — data comes from scraping and in-memory cache.

---

## Folder structure

```
backend/FerryFlight.Api/
├── Controllers/           ← HTTP endpoints (thin layer)
├── Services/              ← Business logic
├── Models/                ← Data shapes (DTOs)
├── Properties/
│   └── launchSettings.json
├── Program.cs             ← App startup, DI, middleware
├── appsettings.json       ← Default config
├── appsettings.Development.json
├── appsettings.Production.json
├── FerryFlight.Api.csproj ← NuGet packages, target framework
├── FerryFlight.Api.http   ← Manual API test file (IDE)
├── Dockerfile             ← Container build for Cloud Run
└── .dockerignore
```

## File reference

### `Program.cs`

**Role:** Application entry point.

**What it does:**

- Registers controllers and JSON camelCase serialization
- Registers `HttpClient` for ferry site and OpenRouteService
- Registers services: `FerryScheduleScraper`, `FerryScheduleCache`, `TravelTimeService`
- Configures CORS from `Cors:Origins` in config
- Reads `PORT` env var for Cloud Run (`http://0.0.0.0:8080`)
- Maps controller routes

> **Best practice:** Keep `Program.cs` thin — wire up DI and middleware only; put logic in Services.

---

### Controllers

#### `Controllers/SchedulesController.cs`

| Item | Value |
|---|---|
| Route | `GET /api/schedules/ferry` |
| Depends on | `IFerryScheduleCache` |

Returns ferry schedule JSON or `503` if scrape fails and no stale cache exists.

#### `Controllers/EstimationController.cs`

| Item | Value |
|---|---|
| Route | `GET /api/estimation/travel-times?latitude=&longitude=` |
| Depends on | `ITravelTimeService` |

Validates coordinates (-90..90 lat, -180..180 lng), returns travel time to Langkawi terminal.

> **Best practice:** Controllers should not contain scraping or caching logic — delegate to services.

---

### Services

#### `Services/IFerryScheduleScraper.cs` + `FerryScheduleScraper.cs`

**Role:** Fetch and parse ferry schedules from the Langkawi Ferry Line website.

- URL: `https://ticket.langkawiferryline.com/index_ext.php?public_trip_info`
- Uses **HtmlAgilityPack** to parse HTML tables
- Extracts up to **3 days**, each with route columns and trip times
- Parses date headers, route names, departure times, vessel names, extra-trip `*` flag

#### `Services/IFerryScheduleCache.cs` + `FerryScheduleCache.cs`

**Role:** Cache layer in front of the scraper.

- Uses `IMemoryCache` with key `ferry-schedules`
- TTL from config: `FerrySchedule:CacheMinutes` (default 30)
- Keeps a **stale copy** in memory — if scrape fails, returns last good data
- Wraps response in `FerryScheduleResponse` with `fetchedAt` timestamp

> **Best practice:** Cache + stale fallback prevents total outage when the external site is slow or down.

#### `Services/ITravelTimeService.cs` + `TravelTimeService.cs`

**Role:** Calculate drive time from user coordinates to Langkawi ferry terminal.

- Destination: configurable lat/lng (default terminal coordinates)
- If `OpenRouteService:ApiKey` is set → call ORS driving directions API
- On failure or missing key → **haversine** straight-line estimate with assumed speed
- Caches results 5 minutes per rounded lat/lng

#### `Services/HaversineTravelEstimator.cs`

**Role:** Math helper for distance-based time estimate when ORS is unavailable.

- Haversine formula for great-circle distance
- Assumes average driving speed
- Formats duration as human-readable text (`"45 min"`, `"1 hr 20 min"`)

### Models

| File | Purpose |
|---|---|
| `Models/FerryScheduleResponse.cs` | Top-level API response: `{ fetchedAt, days[] }` |
| `Models/DaySchedule.cs` | One day: date, labels, routes[] |
| `Models/RouteSchedule.cs` | One route: name, trips[] |
| `Models/Trip.cs` | One sailing: departureTime, vesselName, isExtraTrip |
| `Models/TravelTimeModels.cs` | Travel API request/response types |

All models are simple POCOs (Plain Old C# Objects) — no ORM entities because there is no database.

---

### Configuration files

#### `appsettings.json`

Base settings: logging, cache minutes, CORS localhost, terminal coordinates, empty ORS key.

#### `appsettings.Production.json`

Production overrides (e.g. CORS origins for deployed web URL).

#### `appsettings.Development.json`

Development-specific settings.

**Environment variable override pattern (Cloud Run):**

```
Cors__Origins__0=https://grab-them-all.shariflab.my
OpenRouteService__ApiKey=your-key
```

Double underscore `__` maps to nested JSON keys in ASP.NET Core.

---

### `Dockerfile`

Multi-stage build:

1. **build** — `dotnet restore` + `dotnet publish` with SDK image
2. **final** — copy published output to aspnet runtime image, expose 8080

> **Best practice:** Multi-stage Dockerfiles keep production images small (no SDK in final image).

Defines:

- Target framework: .NET 10

## Backend dependencies (conceptual)

```
SchedulesController
    └── FerryScheduleCache
            └── FerryScheduleScraper
                    └── HttpClient ("FerrySchedule")

EstimationController
    └── TravelTimeService
            ├── HttpClient ("OpenRouteService")
            ├── IMemoryCache
            └── HaversineTravelEstimator (static)
```

All services are registered as **Scoped** in `Program.cs` (new instance per HTTP request).
