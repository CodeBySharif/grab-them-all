# 10. Custom Domain and DNS

This project uses a custom domain with three layers:

```
Exabytes (registrar)  →  Cloudflare (DNS + SSL)  →  GCP Cloud Run (hosting)
```

**Production domains:**

| Domain | Cloud Run service |
|---|---|
| `grab-them-all.shariflab.my` | `ferry-flight-web` |
| `grab-them-all-api.shariflab.my` | `ferry-flight-api` |

---

## Full DNS flow diagram

```
User types: grab-them-all.shariflab.my
        │
        ▼
┌───────────────────┐
│  Exabytes         │  Domain registered: shariflab.my
│  (Registrar)      │  Nameservers → point to Cloudflare
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│  Cloudflare       │  DNS records:
│  (DNS + CDN)      │  grab-them-all      CNAME → ghs.googlehosted.com
│                   │  grab-them-all-api  CNAME → ghs.googlehosted.com
│                   │  SSL: Full or Full (strict)
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│  GCP Cloud Run    │  Domain mappings verified
│  ferry-flight-web │  ← grab-them-all.shariflab.my
│  ferry-flight-api │  ← grab-them-all-api.shariflab.my
└───────────────────┘
```

## Step 1: Domain at Exabytes (registrar)

1. Purchase/register `shariflab.my` (or your domain) at [Exabytes](https://www.exabytes.my/) or your registrar
2. You will manage **DNS records in Cloudflare**, not Exabytes — so you need to delegate nameservers

**Do not** create A/CNAME records for subdomains at Exabytes if Cloudflare manages DNS.

---

## Step 2: Add domain to Cloudflare

1. Sign up / log in at [Cloudflare](https://dash.cloudflare.com/)
2. **Add a site** → enter `shariflab.my`
3. Cloudflare scans existing DNS (optional)
4. Cloudflare gives you **two nameservers**, e.g.:
   - `ada.ns.cloudflare.com`
   - `bob.ns.cloudflare.com`

---

## Step 3: Point Exabytes nameservers to Cloudflare

In **Exabytes control panel**:

1. Go to domain management → `shariflab.my`
2. Find **Nameservers** / DNS delegation
3. Replace default Exabytes nameservers with Cloudflare nameservers
4. Save — propagation can take up to 24–48 hours (often faster)

Verify in Cloudflare dashboard: site status should become **Active**.

---

## Step 4: Map custom domains in GCP Cloud Run

### Web service

1. [Cloud Console](https://console.cloud.google.com/) → **Cloud Run**
2. Select `ferry-flight-web` → **Manage custom domains**
3. **Add mapping** → `grab-them-all.shariflab.my`
4. GCP shows DNS records to create (usually CNAME to `ghs.googlehosted.com`)

### API service

1. Select `ferry-flight-api` → **Manage custom domains**
2. **Add mapping** → `grab-them-all-api.shariflab.my`
3. Note the CNAME target GCP provides

Wait until GCP shows domain status **Active** (certificate provisioning may take 15–30 minutes).

---

## Step 5: Create DNS records in Cloudflare

In Cloudflare → `shariflab.my` → **DNS** → **Records**:

| Type | Name | Target | Proxy |
|---|---|---|---|
| CNAME | `grab-them-all` | `ghs.googlehosted.com` (from GCP) | Proxied 🟠 |
| CNAME | `grab-them-all-api` | `ghs.googlehosted.com` (from GCP) | Proxied 🟠 |

**Use the exact target** GCP gives you for each mapping.

### SSL/TLS settings

Cloudflare → **SSL/TLS** → **Overview**:

- Use **Full** or **Full (strict)**
- Avoid **Flexible** (can cause redirect loops with Cloud Run)

---

## Important: subdomain depth and SSL

Use **single-level** subdomains:

| Domain | Works with Cloudflare free SSL? |
|---|---|
| `grab-them-all.shariflab.my` | ✅ Yes (`*.shariflab.my`) |
| `grab-them-all-api.shariflab.my` | ✅ Yes |
| `api.grab-them-all.shariflab.my` | ❌ No (nested — breaks free SSL) |

This project uses `grab-them-all-api.shariflab.my` for the API for this reason.

## How the browser reaches the API

Users only visit the **web** domain. The browser never needs to call the API domain directly for normal use.

```
Browser → grab-them-all.shariflab.my/api/schedules/ferry
              │
              ▼
         nginx (web container)
              │ proxy_pass BACKEND_URL
              ▼
         grab-them-all-api.shariflab.my/api/schedules/ferry
              │
              ▼
         ASP.NET Core API
```

### Why nginx Host header matters

Cloud Run routes by **Host** header. nginx must send the **API hostname**, not the web hostname:

```nginx
proxy_set_header Host ${BACKEND_HOST};   # grab-them-all-api.shariflab.my
```

Set in `frontend/docker-entrypoint.sh` from `BACKEND_URL` env var.

**Files involved:**

| File | Role |
|---|---|
| `frontend/docker-entrypoint.sh` | Extract `BACKEND_HOST` from URL |
| `frontend/nginx.conf.template` | Proxy rules + Host header |
| `.github/workflows/deploy.yml` | Sets `BACKEND_URL` on deploy |

---

## CI/CD domain configuration

In `.github/workflows/deploy.yml`:

```yaml
PUBLIC_WEB_URL: https://grab-them-all.shariflab.my
PUBLIC_API_URL: https://grab-them-all-api.shariflab.my
```

Deploy sets:

- Web: `BACKEND_URL=${PUBLIC_API_URL}`
- API: `Cors__Origins__0=${PUBLIC_WEB_URL}`

## Verification checklist

After DNS propagates and GCP domains are Active:

```bash
# API direct
curl https://grab-them-all-api.shariflab.my/api/schedules/ferry

# Web proxy (should return same JSON)
curl https://grab-them-all.shariflab.my/api/schedules/ferry

# Web app
curl -I https://grab-them-all.shariflab.my/
```

Expected: JSON from API endpoints, `200` from web root.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Error 525 (SSL) | GCP cert not ready or SSL mode wrong | Wait for Active; use Full SSL |
| 404 on `/api` via web | Wrong `BACKEND_URL` or Host header | Check env + redeploy web |
| 502 on `/api` | API domain unreachable | Check API DNS + Cloud Run mapping |
| API works, web icon wrong | Cloudflare cache | Purge cache for `/sw.js`, icons |
| Nested subdomain SSL fail | Too deep for free Cloudflare SSL | Use flat subdomain |

### Purge Cloudflare cache

After deploys affecting `sw.js`, manifest, or icons:

Cloudflare → **Caching** → **Purge Everything** (or purge specific URLs)

## Summary: who does what

| Layer | Responsibility |
|---|---|
| **Exabytes** | Own the domain; point nameservers to Cloudflare |
| **Cloudflare** | DNS records, SSL edge, CDN cache, DDoS protection |
| **GCP Cloud Run** | Run containers, issue origin SSL for custom domains |
| **GitHub Actions** | Deploy images, set `BACKEND_URL` and CORS |
| **nginx (web)** | Serve PWA, proxy `/api` to backend with correct Host |
