# 7. PWA, Mobile, and Responsive Design

## What is a PWA?

A **Progressive Web App** is a website that can:

- Be **installed** on home screen (like an app icon)
- Work **offline** (partially — cached schedules)
- Run **full screen** without browser chrome (`display: standalone`)

This project uses **vite-plugin-pwa** to generate the manifest and service worker at build time.

---

## PWA files and roles

| File | Role |
|---|---|
| `frontend/vite.config.ts` | PWA plugin config: manifest, workbox, precache |
| `frontend/scripts/generate-pwa-icons.mjs` | Converts `boat.svg` → PNG icons at build time |
| `frontend/public/pwa-192x192.png` | Manifest icon (192×192) |
| `frontend/public/pwa-512x512.png` | Manifest icon (512×512) |
| `frontend/public/maskable-512x512.png` | Android adaptive icon (safe zone padding) |
| `frontend/index.html` | Meta tags, apple-touch-icon, viewport-fit |
| `dist/manifest.webmanifest` | Generated manifest (name, icons, theme, start_url) |
| `dist/sw.js` | Generated service worker |
| `frontend/nginx.conf.template` | Cache headers for SW and static assets |

## Web app manifest

Generated in `vite.config.ts`:

```javascript
manifest: {
  id: '/',
  name: 'Ferry & Flight',
  short_name: 'FerryFlight',
  display: 'standalone',
  start_url: '/',
  theme_color: '#010102',
  background_color: '#010102',
  icons: [ /* 192, 512 any, 512 maskable */ ]
}
```

**Why `id: '/'`:** Helps Chrome treat reinstalls as the same app (important for Android install).

## Service worker strategy

Configured in `vite.config.ts` → `workbox`:

| Setting | Value | Why |
|---|---|---|
| `skipWaiting: true` | Activate new SW immediately | Avoid stuck "installing" on Android |
| `clientsClaim: true` | SW controls open tabs | Faster takeover after deploy |
| `globPatterns` | `index.html`, `assets/*.{js,css}` | Minimal precache — faster install |
| `includeManifestIcons: false` | Icons not precached | Loaded on demand; smaller SW install |
| `navigateFallbackDenylist` | `/^\/api\//` | API calls never fall back to index.html |

### Runtime caching

Ferry schedules API uses **NetworkFirst**:

- Try network first (fresh data)
- Fall back to cache if offline
- Cache name: `ferry-schedules`, max 1 entry, 1 hour TTL

## Service worker registration

Inline in generated `index.html`:

```javascript
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/sw.js', { scope: '/' })
  })
}
```

Simple registration without `workbox-window` — fewer failure points on Android install.

## nginx rules for PWA (production)

From `nginx.conf.template`:

| Path | Behavior |
|---|---|
| `/manifest.webmanifest` | Correct MIME type, no long cache |
| `/sw.js`, `/workbox-*.js` | `Cache-Control: no-cache, must-revalidate` |
| `*.png`, `*.svg` | `try_files` → 404 if missing (no SPA fallback) |
| `/api/*` | Proxy to backend |
| `/*` | SPA fallback to `index.html` |

> **Best practice:** Never use `Cache-Control: no-store` on service worker files — Chrome can hang during install. Never serve HTML for missing PNG paths (breaks iOS icons).

## Mobile-friendly design

### Viewport and safe areas (`index.html` + `index.css`)

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
```

CSS utilities:

| Class | Purpose |
|---|---|
| `.safe-pt` | Padding for iPhone notch / Dynamic Island |
| `.safe-px` | Horizontal safe area (landscape) |
| `.pt-header` | Main content below fixed header + notch |
| `.pb-safe-main` | Bottom padding for home indicator + alert toasts |

### Layout

- **TopBar** — fixed at top with safe-area padding
- **Schedule grid** — 1 column on phone, 3 columns on desktop (`lg:grid-cols-3`)
- **Priority toasts** — fixed bottom with safe-area inset
- **Touch targets** — buttons sized for fingers (`px-3 py-1.5` minimum)

### iPhone-specific

- `apple-mobile-web-app-capable` — full-screen when installed
- `apple-mobile-web-app-status-bar-style: black-translucent` — blends status bar
- Separate **Enable Location** and **Enable Alerts** buttons (iOS requires direct taps)

---

## Desktop-friendly design

Same app, wider layout:

- 3 day cards side by side
- `max-w-7xl` container — readable line length
- Hover states on buttons (`hover:bg-primary-hover`)

Uses Tailwind responsive prefixes — no separate desktop codebase.

## Install flow

### iOS (Safari)

1. Open site in Safari
2. Share → **Add to Home Screen**
3. Icon from `apple-touch-icon` / `pwa-192x192.png`
4. Opens standalone (no Safari UI)

### Android (Chrome)

1. Open site in Chrome
2. Menu → **Install app** or **Add to Home screen**
3. Uses manifest icons (192 + 512 PNG)
4. Should complete in ~10–15 seconds

**If install hangs:** Clear site data, purge Cloudflare cache for `/sw.js`, retry.

## Icon pipeline

```
boat.svg (public/)
    ↓  npm run build
generate-pwa-icons.mjs (sharp)
    ↓
pwa-192x192.png, pwa-512x512.png, maskable-512x512.png
    ↓
Copied to dist/ → Docker image → Cloud Run
```

> **Best practice:** Use **PNG** for PWA manifest icons, not SVG. Android install requires PNG. Use single-level subdomains for API (`grab-them-all-api.domain.com`) — nested subdomains break Cloudflare free SSL.
