## Entry and root

### `index.html`

HTML shell loaded once. Contains:

- Viewport with `viewport-fit=cover` (iPhone notch)
- PWA meta tags (`theme-color`, `apple-mobile-web-app-capable`)
- Icon links (`pwa-192x192.png`, apple-touch-icon)
- Script tag for Vite module entry (`/src/main.tsx`)

### `src/main.tsx`

Mounts React into `#root` with `StrictMode`.

### `src/App.tsx`

**Root component.** Responsibilities:

- Load ferry schedules on mount (`fetchFerrySchedules`)
- Manage loading/error state
- Toggle Ferry vs Flight tab (flight is placeholder)
- Wrap content in `EstimationProvider`
- Layout: `TopBar` + `main` with `EstimationBanner`, `ScheduleGrid`, `PriorityAlerts`

### `src/index.css`

- Tailwind import and `@theme` color tokens (dark theme)
- Safe-area utility classes: `.safe-pt`, `.safe-px`, `.pt-header`, `.pb-safe-main`
- `100dvh` min-height for mobile browsers

## API layer (`src/api/`)

| File | Function | Endpoint |
|---|---|---|
| `schedules.ts` | `fetchFerrySchedules()` | `GET /api/schedules/ferry` |
| `estimation.ts` | `fetchTravelTimes(lat, lng)` | `GET /api/estimation/travel-times` |

Both use **relative URLs** (`/api/...`). In dev, Vite proxies to localhost:5000. In production, nginx proxies to the API service.

> **Best practice:** Use relative API paths so the same build works in dev, Docker, and Cloud Run without embedding backend URLs in the frontend.

## Context (`src/context/`)

### `EstimationContext.tsx`

Shared state for location and travel estimation:

| State / method | Purpose |
|---|---|
| `state` | `idle` \| `locating` \| `loading` \| `ready` \| `denied` \| `error` |
| `travelSeconds` | Drive time in seconds from API |
| `requestLocation()` | Trigger browser geolocation ‚Üí fetch travel times |
| `evaluateTrip()` | Yes/No for a specific trip |
| `notificationPermission` | Browser notification permission status |

Wraps the app in `EstimationProvider`; components use `useEstimation()` hook.

|---|---|
