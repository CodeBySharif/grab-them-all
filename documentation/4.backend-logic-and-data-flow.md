# 4. Backend Logic and Data Flow

## Task 1: Serve ferry schedules

### Sequence diagram

```
Browser                nginx (web)           API                  Ferry website
   │                       │                  │                        │
   │ GET /api/schedules/ferry                 │                        │
   │──────────────────────►│                  │                        │
   │                       │ GET /api/schedules/ferry                 │
   │                       │─────────────────►│                        │
   │                       │                  │ Cache hit?             │
   │                       │                  │── yes ──► return JSON  │
   │                       │                  │                        │
   │                       │                  │ Cache miss:            │
   │                       │                  │ GET schedule HTML      │
   │                       │                  │───────────────────────►│
   │                       │                  │◄───────────────────────│
   │                       │                  │ Parse HTML → JSON      │
   │                       │                  │ Store cache (30 min)   │
   │                       │◄─────────────────│                        │
   │◄──────────────────────│                  │                        │
```

### Step-by-step

1. **Request arrives** at `SchedulesController.GetFerrySchedules()`
2. **Cache check** — `FerryScheduleCache.GetSchedulesAsync()`
   - If cached and not expired → return immediately
3. **Scrape** — `FerryScheduleScraper.ScrapeAsync()`
   - HTTP GET to Langkawi Ferry Line public trip info page
   - Parse HTML tables with class `h5-date` for dates
   - Parse route columns (`div-daily`) for route name + trip rows
   - Each trip: time from `<strong>`, vessel from sibling text
4. **Build response** — `FerryScheduleResponse(utcNow, days)`
5. **Store cache** — 30 minutes (configurable)
6. **Save stale copy** — if next scrape fails, serve this copy
7. **Return JSON** — camelCase property names

### Error handling

| Situation | Response |
|---|---|
| Scrape succeeds | `200` + JSON |
| Scrape fails, stale exists | `200` + stale JSON (logged as warning) |
| Scrape fails, no stale | `503` + `{ message: "..." }` |

### Response shape (simplified)

```json
{
  "fetchedAt": "2026-06-19T04:00:00Z",
  "days": [
    {
      "date": "2026-06-19",
      "dayLabel": "THU",
      "dateLabel": "19 JUN 2026",
      "routes": [
        {
          "name": "Langkawi to Kuala Kedah",
          "trips": [
            {
              "departureTime": "8:00 AM",
              "vesselName": "LANGKAWI FERRY LINE",
              "isExtraTrip": false
            }
          ]
        }
      ]
    }
  ]
}
```

## Task 2: Estimate travel time

### Sequence diagram

```
Browser                          API                    OpenRouteService
   │                              │                            │
   │ GET /api/estimation/travel-times?lat=&lng=               │
   │─────────────────────────────►│                            │
   │                              │ Validate coordinates       │
   │                              │ Cache hit? → return        │
   │                              │                            │
   │                              │ If API key set:            │
   │                              │ GET /directions/driving-car│
   │                              │───────────────────────────►│
   │                              │◄───────────────────────────│
   │                              │                            │
   │                              │ Else / on failure:         │
   │                              │ Haversine estimate         │
   │                              │ Cache 5 min                │
   │◄─────────────────────────────│                            │
```

### Step-by-step

1. **Request** — `EstimationController.GetTravelTimes(latitude, longitude)`
2. **Validate** — lat ∈ [-90, 90], lng ∈ [-180, 180]
3. **Cache key** — `travel-{lat rounded}-{lng rounded}`
4. **Try OpenRouteService** (if API key configured):
   - Driving directions from user → Langkawi terminal
   - Parse `features[0].properties.summary.duration` (seconds)
   - Provider: `"openrouteservice"`
5. **Fallback — Haversine**:
   - Great-circle distance × assumed road speed
   - Provider: `"haversine"` (approximate)
6. **Response**:

```json
{
  "destination": { "latitude": 6.3197, "longitude": 99.8431 },
  "langkawiTerminal": {
    "durationSeconds": 1800,
    "durationText": "30 min",
    "provider": "openrouteservice"
  }
}
```

## Caching strategy

| Data | Cache | TTL | Stale fallback |
|---|---|---|---|
| Ferry schedules | `IMemoryCache` | 30 min | Yes (in-memory stale copy) |
| Travel times | `IMemoryCache` | 5 min | No |

> **Best practice:** Cache expensive external calls. Use stale fallback only for data that changes slowly (schedules), not for user-specific estimates.

---

## Why scrape instead of an official API?

The Langkawi Ferry Line site publishes schedules as HTML, not JSON. Scraping is fragile (site layout changes can break the parser) but is the only option without an official API partnership.

**Mitigations in this project:**

- HtmlAgilityPack with specific CSS/class selectors
- Stale cache fallback
- Logging when parse fails
- 503 error instead of empty/broken data
